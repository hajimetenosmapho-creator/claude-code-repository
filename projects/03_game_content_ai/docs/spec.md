# 03_game_content_ai 仕様書

作成日：2026-06-24  
バージョン：MVP (v1)

## ツール概要

国内外のゲームニュースをRSSで自動収集し、Claude AIが重要度を判定・日本語記事下書き・SEOタイトル・X投稿文を生成して、Markdownファイルとして保存するツール。

**ブログ名**：KAORUの部屋  
**対象読者**：日本のゲームユーザー（初心者〜中級者）

---

## ニュース収集元（MVP対象）

| サイト | 言語 | 取得方法 |
|--------|------|----------|
| 4Gamer | 日本語 | RSS |
| Game*Spark | 日本語 | RSS |
| IGN | 英語 | RSS |
| GameSpot | 英語 | RSS |
| Eurogamer | 英語 | RSS |

## キーワードフィルター

Claude API呼び出し前に、取得した全ニュースをキーワードでスクリーニングしてAPI使用量を削減する。

### 優先キーワード（含む場合はフィルター通過）

```
Nintendo, Switch, PlayStation, PS5, PS6, Xbox,
controller, gaming device, accessory, review,
任天堂, プレイステーション, コントローラー, ゲーミングデバイス, 周辺機器
```

### 低優先・除外の判定

| 判定 | 条件 | 処理 |
|------|------|------|
| 通過 | 優先キーワードにヒット | 重要度判定へ進む |
| 保留 | ヒットなしだがゲーム関連の可能性あり | 低優先としてリスト末尾へ |
| 除外 | 明らかに無関係 | ログに記録してスキップ |

除外対象の例：
- 純粋なeスポーツ大会結果（ゲーム機関連でない）
- 映画・ドラマのゲーム原作報道（ゲームプレイ情報なし）
- モバイルゲームのみの話題（コンソール・PC無関係）

---

## 重要度分類

AIが以下の基準で重要度を判定する。

| 重要度 | 基準 | 記事化方針 |
|--------|------|-----------|
| **S** | Switch後継機・PS5/PS6・Xbox次世代機・新型ゲーム機/コントローラー/デバイス・大手メーカー発表 | 優先して記事化（2000文字以上） |
| **A** | 大型タイトル発表・人気DLC・大型アップデート・発売日/価格/予約情報 | 通常記事化（800〜1500文字） |
| **B** | セール・小規模アップデート・周辺ニュース | 必要に応じて短文記事化 |

**ブログ重点テーマ**：ゲーム周辺機器・コントローラー・Nintendo Switch・PlayStation

---

## 記事スタイル

| ニュース種別 | 文字数 | 構成要素 |
|-------------|--------|---------|
| 通常（重要度A） | 800〜1500文字 | ニュース概要・何が変わるか・ユーザーへの影響・考察 |
| 大型（重要度S） | 2000文字以上 | 上記＋メリット/デメリット・おすすめできる人 |
| 速報のみ（価格・セール） | 短め | 要点のみ簡潔に |

単純翻訳ではなく、日本のゲームユーザー向けに分かりやすく解説するブログ記事にする。

---

## KAORUの部屋 ブログ方針

記事生成時にAIへ必ず伝える方針。

**文章スタイル**
- ゲーム初心者にも分かりやすい言葉を使う
- ゲームユーザー目線（消費者として正直に）
- 専門用語は初出時に括弧で説明を入れる（例：4K（超高解像度映像）対応）
- メリット・デメリットを両方書く（良いことだけ書かない）
- 購入判断を助ける内容を意識する（「こんな人におすすめ」「こんな人は待った方がいい」）

**重点ジャンル**（これらのニュースは特に丁寧に書く）
- Nintendo Switch / Switch後継機
- PlayStation / PS5 / PS6
- Xbox
- コントローラー
- ゲーム周辺機器・ゲーミングデバイス

---

## SNS反応の扱いルール

**禁止**
- 実際に取得していないSNSやRedditの反応をAIが創作すること
- 「SNSで話題」「海外ユーザーから絶賛」など根拠のない表現

**代替表現**（取得できない場合）
- 「今後ユーザーの反応が注目されます」
- 「発売後のユーザー評価に期待が集まります」
- 「コミュニティの反応は今後注目されるでしょう」

SNS・コミュニティ反応を書く場合は、実際に取得したデータのみ使用する。

## SEO方針

| ニュース種別 | 方針 |
|-------------|------|
| 速報（価格・発売日・セール） | 短め・早め（他サイトより先に公開を意識） |
| 大型ニュース（S級） | 詳しく書いて検索上位を狙う |

SEOタイトルには対象ゲーム機名・タイトル名などのキーワードを含める。

## X（旧Twitter）投稿スタイル

**通常ニュース**
```
[興味を引く一言]
ポイント👇
✅ [重要ポイント1]
✅ [重要ポイント2]
✅ [発売日・価格など]

詳しくまとめました👇
[ブログURL]
```

**大型ニュース**
```
[読者が気になる疑問から始める一言]

[内容の紹介：旧モデルとの違い・メリット・デメリット・おすすめできる人]

[ブログURL]
```

過度な煽りや釣りタイトルは避け、ゲーム好きが読みたくなる自然な投稿にする。

## 画像方針（MVP）

記事には画像を掲載しない（テキストのみ）。  
ただし、将来的な画像対応に備えて以下のフィールドをMarkdownメタデータに保存する。

| フィールド | 内容 |
|-----------|------|
| `official_news_url` | 公式ニュースURL |
| `official_site_url` | 公式サイトURL |
| `official_trailer_url` | 公式YouTubeトレーラーURL |
| `official_presskit_url` | 公式プレスキットURL |
| `image_candidates` | 画像候補URLのリスト |
| `image_source` | 画像提供元 |
| `image_terms_confirmed` | 利用条件確認済みかどうか |

記事生成時に画像候補として表示するだけ。実際の掲載は人間が確認して選択する。  
v2以降で、著作権的に問題ない公式素材のみ利用できる機能を追加予定。

## 実行方法（MVP）

```bash
python main.py
```

手動実行で起動する。動作確認・エラー対応・記事確認をしてから手動でWordPressへコピーする。

**v2以降の予定**：Windowsタスクスケジューラによる定時自動実行（投稿は人間確認後）

---

## データフロー

```
1. main.py 起動
   ↓
2. collector.py：各サイトのRSSを取得 → ニュース一覧
   ↓
3. keyword_filter.py：キーワードで事前スクリーニング（API節約）
   ↓
4. importance_judge.py：Claude APIで重要度(S/A/B)を判定
   ↓
5. article_generator.py：重要度に応じた記事下書きを生成
   ↓
6. seo_title_generator.py：SEOタイトルを生成
   ↓
7. x_post_generator.py：X投稿文を生成
   ↓
8. output/ にMarkdownファイルとして保存
   ↓
9. ターミナルに生成結果サマリーを表示
```
