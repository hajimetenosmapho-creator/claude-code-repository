## 1. プロジェクト概要

このリポジトリは、Claude Code を活用して構築する **個人用AIツール群の開発基盤** です。

ブログ運営・YouTube制作・ゲームニュース配信・投資管理・消防救急業務支援など、日常の仕事や創作活動における繰り返し作業をAIが担うことで、人間はより価値の高い判断・創造・表現に集中できる環境を目指します。

複数のAIツールを個別に作るのではなく、**共通の設計思想・構成・開発ルールのもとで一体的に育てていく** ことがこのプロジェクトの本質です。

---

## 2. 基本理念

### AIは人の代わりではなく、人を支援するもの

AIは人間の仕事を奪うものではありません。繰り返し・単純・時間のかかる作業をAIに任せ、人がより価値の高い判断や創造に集中できるよう支援するために活用します。

### 小さく作り、小さく改善し、長く育てる

完璧なシステムを一度に作ろうとしない。動く最小単位から始め、使いながら少しずつ改善していく。このリポジトリは完成するものではなく、継続的に育てていくものです。

### 可読性・保守性・再利用性を重視する

数年後の自分や別の環境でも読んで理解できるコードとドキュメントを書く。コードは動けばよいのではなく、読みやすく・直しやすく・再利用しやすいものを目指します。

## 3. 開発方針

### ChatGPT と Claude Code の協業

| 役割 | 担当 |
|---|---|
| 設計・アーキテクチャ | ChatGPT |
| 技術選定・比較検討 | ChatGPT |
| レビュー・リリース判断 | ChatGPT |
| 長期計画・方向性の決定 | ChatGPT |
| 実装・リファクタリング | Claude Code |
| テスト・動作確認 | Claude Code |
| Git 操作（commit / push） | Claude Code |
| ドキュメント更新 | Claude Code |

### ドキュメントはコードと同じくらい重要

実装と同時に README・CHANGELOG・ROADMAP を更新する。ドキュメントが整っていることが、長期運用と品質の基盤です。

---

## 4. 品質基準

- **動くだけでは完成ではない**：保守できなければ完成ではない
- **保守しやすいこと**：後から変更・拡張しやすい構成にする
- **読みやすいこと**：難しい書き方より、明快なコードを選ぶ
- **小さく実装し、小さくテストすること**：動作確認できる単位で進め、一度に大きく変えない
- **実測主義**：確認できていないことを推測で報告しない。確認できない場合は「確認できませんでした」と正直に報告する

---

## 5. Git 運用

```
# 1. 最新状態を取得
git pull

# 2. 機能ブランチで作業（大きな変更の場合）
git checkout -b feature/xxx

# 3. 変更をコミット（意味のある単位で）
git add <ファイル>
git commit -m "feat/fix/docs: 変更内容の説明"

# 4. push 前に確認
git status            # clean であること
git log --oneline     # コミット内容を確認

# 5. GitHub へ push
git push origin main（または feature/xxx）

# 6. マージ後はブランチを削除
git branch -d feature/xxx
```

**絶対に守るルール：**
- `.env`・APIキー・パスワードはコミットしない
- push 前に working tree が clean であることを確認する

## 8. 今後のビジョン

このプロジェクトが目指す将来像は、個別のAIツールの集合体ではなく、**一つのAI開発基盤の上で複数の生活・仕事領域を支援するエコシステム** です。

**ブログ運営の効率化**  
ゲームニュース収集から記事生成・WordPress投稿まで自動化。読者に価値ある情報を、人間の目で最終確認しながら届ける。

**YouTube制作支援**  
動画の内容をテキスト化し、ブログ記事・SNS投稿文・概要欄などに展開。一つのコンテンツから複数の媒体へ効率的に発信する。

**ゲームニュース自動配信**  
16サイト以上のRSSから最新情報を収集・判定・記事化。スクープを見逃さず、速報タグ付きで自動下書き。

**投資・家計管理支援**  
収支データをAIが分析し、傾向・注意点・改善案をレポート。感情に流されない判断の補助を行う。

**消防・救急業務支援**  
救急活動記録・公文書の作成をAIが補助。客観的・論理的な文書を効率よく作成し、現場の判断に集中できる環境をつくる。

**AIエージェントの構築**  
各ツールが連携し、スケジュール実行・通知・承認フローを自動化。人間は最終確認と判断のみを行う体制へ。

これらすべてを、共通の設計思想・技術スタック・開発ルールのもとで一体的に育てていきます。

## 9. 最後に

このプロジェクトは、AIでコードを書くことが目的ではありません。

**AIはコードを書くことが目的ではない。**  
**人がより価値のある仕事に集中するための基盤である。**

コードが増えるほど、ドキュメントが整うほど、このリポジトリは「道具」から「基盤」へと育っていきます。小さな一歩を積み重ねながら、長く・丁寧に育てていきましょう。
