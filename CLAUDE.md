## GitHub運用ルール

「GitHubへ保存して」「pushして」「作業内容を保存して」と言われた場合は以下の順番で行う。

1. `git status` で変更確認
2. 変更ファイルを説明
3. 以下が含まれていないか確認
   - `.env`
   - APIキー
   - パスワード
   - 個人情報
   - PC固有設定ファイル（`.claude/settings.local.json` など）
4. 問題なければ `git add .`
5. 内容に合わせた分かりやすいコミットメッセージを作成
6. `git commit`
7. `git push origin main`
8. 結果を初心者向けに説明する

勝手にpushせず、重要な変更時は確認する。

## セキュリティ

以下はGitHubへのアップロード禁止。

- `.env`
- APIキー
- パスワード
- 個人情報CSV
- ログファイル
- `.claude/settings.local.json`

`.env.example`（ひな形）のみ共有可能。
