## 1. スラッシュコマンド一覧

### 基本操作

| コマンド | 説明 |
|---------|------|
| `/help` | ヘルプを表示。使い方の概要とコマンド一覧を確認できる |
| `/clear` | 現在の会話履歴をリセット。新しいコンテキストで再スタート |
| `/exit` | Claude Code を終了 |
| `/quit` | Claude Code を終了（`/exit` と同じ） |

### 設定・情報

| コマンド | 説明 |
|---------|------|
| `/config` | 設定を変更。モデル・テーマ・言語などを対話式で変更できる |
| `/cost` | 現在の会話で消費したトークン数と推定コストを表示 |
| `/status` | 現在の設定状態（使用モデル・権限モードなど）を表示 |
| `/fast` | Fastモード（Claude Opus + 高速出力）をトグルON/OFF |

### コード品質・レビュー

| コマンド | 説明 |
|---------|------|
| `/init` | プロジェクトを分析してCLAUDE.mdを自動生成 |
| `/review` | GitHubのプルリクエストをレビュー（PR番号を引数で渡す） |
| `/code-review` | 現在のdiffをコードレビュー。オプション: `low` / `medium` / `high` / `ultra` |
| `/code-review ultra` | クラウド上でマルチエージェントによる深いレビューを実行 |
| `/code-review --fix` | レビュー結果を自動でコードに適用 |
| `/code-review --comment` | レビュー結果をPRのインラインコメントとして投稿 |
| `/simplify` | 変更コードをシンプル化・効率化する提案と自動適用 |
| `/security-review` | 現在ブランチの変更に対してセキュリティレビューを実施 |

### 自動化・スケジューリング

| コマンド | 説明 |
|---------|------|
| `/schedule` | クラウドエージェントをcronスケジュールで定期実行する設定 |
| `/loop [間隔] [コマンド]` | 指定間隔でコマンドを繰り返し実行（例: `/loop 5m /code-review`） |
| `/loop` | 間隔省略でモデルが自分でペースを判断して繰り返す |

### メモリ・コンテキスト管理

| コマンド | 説明 |
|---------|------|
| `/memory` | 保存されているメモリ（記憶）を確認・管理 |

### その他スキル

| コマンド | 説明 |
|---------|------|
| `/run` | アプリを起動して動作確認。変更が実際に動くか確認する |
| `/verify` | コード変更が正しく動作しているかブラウザ等で検証 |
| `/update-config` | settings.json を更新してフック・権限・環境変数を設定 |
| `/keybindings-help` | キーバインドのカスタマイズ方法をガイド |
| `/fewer-permission-prompts` | よく使うコマンドを分析して権限プロンプトを減らす設定を追加 |
| `/claude-api` | Claude API / Anthropic SDK のリファレンスを参照 |

## 2. キーボードショートカット

### 入力・送信

| ショートカット | 動作 |
|--------------|------|
| `Enter` | メッセージを送信 |
| `Shift + Enter` | 改行（複数行入力） |
| `Escape` | 入力内容をクリア / 実行中の処理をキャンセル |
| `Ctrl + C` | 現在の処理を中断 |

### 履歴ナビゲーション

| ショートカット | 動作 |
|--------------|------|
| `↑` (上矢印) | 前のメッセージ履歴に戻る |
| `↓` (下矢印) | 次のメッセージ履歴に進む |
| `Ctrl + R` | 過去の入力を検索（ターミナル環境） |

### 画面操作

| ショートカット | 動作 |
|--------------|------|
| `Ctrl + L` | 画面をクリア（ターミナル環境） |
| `Tab` | ファイルパスやコマンドの自動補完 |

## 3. CLI起動オプション（フラグ）

コマンドラインから `claude` コマンドに渡せるオプションです。

### 基本フラグ

| フラグ | 短縮形 | 説明 |
|--------|--------|------|
| `--help` | `-h` | ヘルプを表示して終了 |
| `--version` | `-v` | バージョン番号を表示 |
| `--print` | `-p` | 非インタラクティブモード。1回だけ応答して終了 |
| `--verbose` | なし | 詳細なデバッグ情報を出力 |

### モデル・API設定

| フラグ | 説明 |
|--------|------|
| `--model <model-id>` | 使用するモデルを指定（例: `--model claude-sonnet-4-6`） |
| `--api-key <key>` | Anthropic APIキーを指定（`ANTHROPIC_API_KEY`環境変数の代替） |

### 出力形式

| フラグ | 説明 |
|--------|------|
| `--output-format <format>` | 出力形式を指定: `text`（デフォルト）/ `json` / `stream-json` |
| `--no-color` | カラー出力を無効化 |

### 権限・ツール制御

| フラグ | 説明 |
|--------|------|
| `--permission-mode <mode>` | 権限モード: `default` / `accept-edits` / `bypass-permissions` / `plan` |
| `--allowedTools <tools>` | 使用を許可するツールをカンマ区切りで指定 |
| `--disallowedTools <tools>` | 使用を禁止するツールをカンマ区切りで指定 |

### セッション制御

| フラグ | 説明 |
|--------|------|
| `--max-turns <n>` | 非インタラクティブモードでの最大ターン数を指定 |
| `--add-dir <path>` | 追加でアクセス許可するディレクトリを指定 |
| `--resume <session-id>` | 過去のセッションを再開 |

### 使用例

```bash
# 非インタラクティブに1回だけ質問
claude -p "このコードのバグを見つけて"

# モデルを指定して起動
claude --model claude-opus-4-8

# JSONで出力（スクリプト連携に便利）
claude -p "ファイル一覧を教えて" --output-format json

# 読み取り専用モードで起動（ファイル編集不可）
claude --permission-mode plan
```

## 4. エージェントタイプ一覧

`Agent` ツールの `subagent_type` に指定できる専門エージェントです。

| エージェント名 | 用途 |
|--------------|------|
| `claude` | 汎用。どのタスクにも対応するデフォルトエージェント |
| `claude-code-guide` | Claude Code自体の機能・使い方に関する質問に特化 |
| `Explore` | コードベースの探索・検索専用（高速な読み取り専用エージェント） |
| `general-purpose` | 複雑な調査・マルチステップタスク向け汎用エージェント |
| `Plan` | 実装計画の設計・アーキテクチャ検討向けエージェント |
| `statusline-setup` | Claude Codeのステータスライン設定を構成するエージェント |

## 5. モデルID一覧

Claude APIで利用できる最新モデルIDです（2026年6月時点）。

| モデル名 | モデルID | 特徴 |
|---------|----------|------|
| Fable 5 | `claude-fable-5` | 最新・最高性能モデル |

## 6. ターミナル操作Tips

### ファイルをコンテキストに追加する方法

```bash
# ファイルの内容を読ませる
cat src/main.py | claude -p "このコードを改善して"

# 複数ファイルを渡す
claude -p "このプロジェクトを分析して" < README.md
```

### よく使うワークフロー

```bash
# プロジェクト開始時
claude /init          # CLAUDE.mdを自動生成

# コードレビュー
claude /code-review high        # 詳細なレビュー
claude /code-review ultra       # クラウドでの最深レビュー

# 変更を検証してからコミット
claude /verify        # 実際の動作を確認
git add -p            # 変更を確認しながらステージング

# 定期タスクの自動化
claude /schedule      # cronでClaudeを定期実行
```

### 権限モードの使い分け

| モード | 説明 | 使いどころ |
|--------|------|-----------|
| `default` | 危険な操作は確認を求める | 通常使用 |
| `accept-edits` | ファイル編集を自動許可 | 積極的に編集させたいとき |
| `bypass-permissions` | すべての操作を自動許可 | 信頼済み自動化スクリプト内 |
| `plan` | 読み取り専用（編集不可） | コードを変えずに分析だけしたいとき |
